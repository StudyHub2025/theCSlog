## 1. 주문 프로젝트 개요

### 전체 구조
- **도메인 구성**: 파트너, 상품, 주문
- **이해관계자**: 유저, 파트너, 내부 운영자

### 도메인 요구사항
#### 1. 파트너
- 시스템에 등록된 파트너만 상품 등록/주문 처리 가능
- 필수 입력: 파트너명, 사업자번호, 이메일
- 계약 종료 시 비활성화 (삭제 X)
- 가입 완료 시 이메일 알림 발송
- 기본 CRUD 제공

#### 2. 상품
- 활성화된 파트너만 등록 가능
- 옵션: 색상/사이즈 (순서 지정 가능), 추가 가격 적용 가능
- 상태: 판매 준비중, 판매중, 판매 종료
- 수량 등은 단순화를 위해 생략

#### 3. 주문
- 유저가 상품을 선택해 주문 → 결제 → 배송
- 결제수단: 카드, 토스페이, 카카오페이, 네이버페이
- 결제 완료 시 카카오톡 알림
- 배송 상태 단계: 준비중 → 배송중 → 완료
